About This Blog

把学习过程写成以后还能回来的路径

这里主要记录三件事：系统学 Python、按专题推进 LeetCode Hot100、把 Hexo + Butterfly 博客从“能用”慢慢搭到“更顺手”。我不太想只堆答案或配置片段，更想把每一步为什么这样做写清楚。

看 Python 系列 看 Hot100 导航
我在写什么

学习路线比零散文章更重要

这个博客现在最核心的不是“文章数量”，而是把内容逐渐整理成可以顺着走的路径。读者第一次进来时，应该能很快知道先看什么、看完下一步去哪。

适合怎么读

从导航页进，比从归档页进更舒服

如果你是第一次来，建议优先走专题入口：/python-core/、/leetcode-hot100/、/build-blog/。如果只是想随便逛逛，再去归档、标签或随机页面。

这个站怎么搭的

Hexo + Butterfly + GitHub Pages

技术栈偏轻：Hexo 负责静态生成，Butterfly 负责结构与主题，自定义 CSS/JS 负责视觉和交互小改造，最后发布到 GitHub Pages。

当前最推荐的阅读入口

01 Python教程核心章节

把原本很长的教程拆成 5 个更适合复习的核心章节。

 02 LeetCode Hot100

按题型推进，不只是刷题，也记录题感、模板和 Python 写法。

 03 博客搭建与维护

记录从博客初始化、主题配置到 SEO、订阅和长期维护的过程。

我希望这个博客慢慢变成什么样

  • 文章越写越像一套有顺序、有入口的学习地图
  • 每个系列都能让人顺着读，而不是看完一篇就断掉
  • 页面保持克制，但在细节上有一点自己的气质
  • 以后回来看时，仍然能快速找到当时真正有价值的内容
